Layout Types

Example of a Row layout (class="section-row")

The section structure draws inspiration from slide decks, which often showcase fun and interesting layout designs to elevate their content. It also makes it easier to transition case studies into presentation format should the need arise.

Other details about this particular layout:

  • This layout can technically fit more than two columns if its children have a low min-width property. Otherwise they will begin to wrap.
  • Any number of elements will fit in this column, but this example has two paragraph elements.
  • At smaller widths, the right-side column will wrap below.

Content groups (smaller h4 header, same row layout)

This column is a content group.
Content groups are usually made up of a stack of different element types representing a single point or concept. The next layout has an example of a grid-card content group.
Originally, I wanted to use some kind of component equivalent to create a more modular system.
But after looking at available options, it seemed too difficult to take on for a project I wanted to complete relatively soon.
A front-end framework like React seemed overkill, and I also briefly considered MDN's web components before realizing it isn't supported by Safari.

Grid layout (class="section-grid")

Browseable format to showcase multiple pieces of info
A card can contain basically whatever you want, but it looks best when each card has a similar amount of content.
Good for showcasing sticky notes & brief data points
Unlike FigJam screenshots, this will be both easier to read by humans better for screen readers.
Defaults to columns of three
To ensure a comfortable line width for reading, the grid cards have a minimum width of 250px and will change to a 2 or 1-column layout depending on viewport size.
Avoid putting too much crucial information here
Grid cards are best used for multiple bits of smaller, optional information. It shouldn't be useless per se, but make sure the case study still makes sense if readers skip it.
You can also add content above or below a lot of these layouts for supplementary information or to create transitions between sections. This one uses the "markup-circle" and "text-emphasis" classes to draw attention to this important point.
